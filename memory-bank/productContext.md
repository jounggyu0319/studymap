# Product Context

## UX 방향 (확정)

### 메인 대시보드
- 폴더 탭 (학기별 분류: 26-1학기, 25-2학기 등)
- 카드 정렬: 마감 급한 순 / 진척도 낮은 순 / 사용자 지정
- 태그 필터: "사용자 지정" 클릭 시에만 표시 (평소 숨김)

### 카드 구조 (접힌 상태)
- 유형 태그 (시험/과제/독서 등) + 과목명
- 카드 제목
- D-day + 마감 날짜
- 진척도 게이지 + 퍼센트
- "다음 할 일" 한 줄 (항상 표시)

### 카드 클릭 시 상세 화면
- 체크리스트 (채팅 업데이트 결과 반영, 직접 클릭도 가능)
- AI 공부 계획 (마감일 역산 자동 생성)
- 메모장 (자유 입력)
- 별도 채팅창 없음

### 진척도 업데이트 방식
- **메인 채팅창 하나**로 통일 (대시보드 하단 고정)
- "IO 1~3강 봤어" → AI가 카드 목록 전체 컨텍스트로 매칭 → 서브태스크 완료 처리
- 체크박스 직접 클릭도 병행 가능
- 토큰 비용: 카드 5~10개 수준 → 요청당 ~1000토큰 (무시 가능)

### 업로드 플로우
- 업로드 즉시 카드 생성 (검토 단계 없음)
- PDF, 이미지, 텍스트 붙여넣기 지원
- 카드 생성 후 인라인 수정 가능

### 디바이스 전략
- 데스크탑: 업로드/편집 (Control Tower)
- 모바일/태블릿: 조회 + 진척도 체크 (Dashboard)

## 유형 태그 9종
시험 / 퀴즈 / 과제 / 팀플 / 발표 / 실습 / 독서 / 보고서 / 기타

## 진척도 색상
- 0~39%: 빨강
- 40~69%: 주황
- 70~100%: 초록

## 강의계획서 처리 원칙
- 시험 이벤트(중간고사, 기말고사)만 추출
- 정기 과제(Problem Sets, Weekly HW 등) 추출 금지
- 강의계획서는 시험 일정/범위 파악 보조용
